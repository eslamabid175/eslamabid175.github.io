// 01 — الأساسيات

Flutter & Dart Basics

📱
Everything is Widget
في Flutter كل حاجة عبارة عن Widget: layout, text, buttons, animations. ده بيخلي بناء الـ UI modular وسهل إعادة الاستخدام.
🎯
Stateless vs Stateful
StatelessWidget للـ UI الثابت، و StatefulWidget لما يبقى فيه state بيتغير زي loading, form values, pagination.
⚡
Production Bootstrap
اللي اتعلمته فعليًا إن بداية التطبيق أهم من أول screen: لازم تجهز Firebase, Localization, DI, cache, notifications قبل runApp.
🧠
Localization & RTL
في التطبيقات العربية الحقيقية لازم الـ RTL والترجمة يبقوا جزء من التصميم من البداية، مش خطوة متأخرة بعد ما الشاشات تخلص.
💡 أول درس عملي: التطبيق الإنتاجي مش مجرد Widget tree. البداية الصح هي ترتيب startup sequence: bindings → Firebase/services → localization → dependency injection → offline cache → state providers → app shell.

State Management

💡 قاعدة مهمة: اختار state management على حسب حجم المشروع:
  • مشروع صغير: state بسيط داخل الشاشة أو Cubit صغير

  • مشروع كبير: Cubit/BLoC + offline cache + token refresh + tests واضحة

Cubit Pattern من مشروع إنتاجي

dart enum AuthStatus { initial, loading, authenticated, unauthenticated, error } class AuthState extends Equatable { final AuthStatus status; final AuthUser? user; final String? errorMessage; const AuthState({ this.status = AuthStatus.initial, this.user, this.errorMessage, }); AuthState copyWith({AuthStatus? status, AuthUser? user, String? errorMessage}) { return AuthState( status: status ?? this.status, user: user ?? this.user, errorMessage: errorMessage, ); } } class AuthCubit extends Cubit<AuthState> { final AuthRemoteDataSource _dataSource; final SecureStorageService _storage; AuthCubit({required AuthRemoteDataSource dataSource, required SecureStorageService storage}) : _dataSource = dataSource, _storage = storage, super(const AuthState()); Future<void> checkAuth() async { final token = await _storage.getAccessToken(); if (token == null) { emit(const AuthState(status: AuthStatus.unauthenticated)); return; } try { final user = await _dataSource.getProfile(); emit(AuthState(status: AuthStatus.authenticated, user: user)); } catch (_) { await _storage.clearTokens(); emit(const AuthState(status: AuthStatus.unauthenticated)); } } }

Clean Architecture

🧩
Presentation Layer
Widgets + Cubits/BLoCs + UI logic. مسؤول عن عرض البيانات والتفاعل مع المستخدم بس.
🏛️
Domain Layer
Use cases + entities + repository contracts. ده قلب البزنس logic المستقل عن أي framework.
🗄️
Data Layer
Repository implementation + remote/local data sources + models/mappers.

dart class DioClient { late final Dio dio; final FlutterSecureStorage _storage = const FlutterSecureStorage(); DioClient() { dio = Dio( BaseOptions( baseUrl: ApiConstants.baseUrl, connectTimeout: const Duration(seconds: 15), receiveTimeout: const Duration(seconds: 15), headers: {'Accept': 'application/json', 'Accept-Language': 'ar'}, ), ); dio.interceptors.addAll([ _authInterceptor(), LogInterceptor(requestBody: true, responseBody: true, error: true), ]); } InterceptorsWrapper _authInterceptor() { return InterceptorsWrapper( onRequest: (options, handler) async { final token = await _storage.read(key: 'access_token'); if (token != null) { options.headers['Authorization'] = 'Bearer $token'; } handler.next(options); }, onError: (error, handler) async { if (error.response?.statusCode == 401 && await _refreshToken()) { final token = await _storage.read(key: 'access_token'); error.requestOptions.headers['Authorization'] = 'Bearer $token'; final response = await dio.fetch(error.requestOptions); return handler.resolve(response); } handler.next(error); }, ); } }

// 05 — Local Storage & Offline

Local DB & Offline Thinking

🧱
Hive for Cache
استخدمت Hive لتخزين JSON cache, user preferences, timestamps, وآخر بيانات متاحة للمستخدم لما الشبكة تكون ضعيفة أو مقطوعة.
🗃️
sqflite for Structured Data
لما البيانات تبقى relational أو محتاجة CRUD منظم محليًا، sqflite مناسب أكتر من key-value caching.
🔐
Secure Storage
التوكنز وبيانات الجلسة الحساسة لازم تروح لـ flutter_secure_storage مش Hive أو SharedPreferences.
⏱️
TTL + Cleanup
الكاش الناجح مش تخزين فقط. لازم تحدد max age, timestamps, وتنضف البيانات المنتهية بوضوح.

Hive Cache Service

dart class CacheService { static const String _merchantsBoxName = 'merchants_cache'; static const String _userPrefsBoxName = 'user_prefs'; late Box<String> _merchantsBox; late Box<String> _userPrefsBox; Future<void> initialize() async { await Hive.initFlutter(); _merchantsBox = await Hive.openBox<String>(_merchantsBoxName); _userPrefsBox = await Hive.openBox<String>(_userPrefsBoxName); } Future<void> cacheMerchants(String category, List<Map<String, dynamic>> merchants) async { await _merchantsBox.put(category, jsonEncode(merchants)); await _merchantsBox.put('${category}_timestamp', DateTime.now().toIso8601String()); } List<Map<String, dynamic>>? getCachedMerchants(String category, {Duration maxAge = const Duration(hours: 1)}) { final data = _merchantsBox.get(category); final timestampStr = _merchantsBox.get('${category}_timestamp'); if (data == null || timestampStr == null) return null; if (DateTime.now().difference(DateTime.parse(timestampStr)) > maxAge) return null; return (jsonDecode(data) as List).cast<Map<String, dynamic>>(); } }

// 06 — Optimization

Performance Best Practices

🔧
Use const Widgets
استخدام const يقلل rebuilds وبالتالي بيحسن الأداء خاصة في الـ widget trees الكبيرة.
🧵
Isolates
العمليات الثقيلة (parsing كبير, encryption, image processing) يفضل تتنفذ في isolate بعيد عن main thread.
🧹
Dispose Correctly
لازم تعمل dispose للـ controllers والـ streams لتجنب memory leaks مع الوقت.
🖼️
Image Optimization
استخدم lazy loading + caching + أحجام صور مناسبة لتقليل استهلاك الذاكرة وتسريع الـ rendering.

dart Future<List<Item>> parseLargeJson(String jsonString) async { return compute(_parseItems, jsonString); } List<Item> _parseItems(String input) { final decoded = jsonDecode(input) as List; return decoded.map((e) => Item.fromJson(e)).toList(); }

Testing Strategy

Cubit Test من مشروع فعلي

dart void main() { late AuthCubit cubit; late MockAuthDataSource mockDataSource; late MockSecureStorage mockStorage; setUp(() { mockDataSource = MockAuthDataSource(); mockStorage = MockSecureStorage(); cubit = AuthCubit(dataSource: mockDataSource, storage: mockStorage); }); group('AuthCubit', () { test('emits unauthenticated when no token stored', () async { when(() => mockStorage.getAccessToken()).thenAnswer((_) async => null); await cubit.checkAuth(); expect(cubit.state.status, AuthStatus.unauthenticated); }); test('emits authenticated when token valid and profile loads', () async { when(() => mockStorage.getAccessToken()).thenAnswer((_) async => 'valid_token'); when(() => mockDataSource.getProfile()).thenAnswer((_) async => user); await cubit.checkAuth(); expect(cubit.state.status, AuthStatus.authenticated); expect(cubit.state.user, user); }); }); }
اختبار متوازن = Unit tests (المنطق) + Widget tests (سلوك الواجهة) + Integration tests (رحلة المستخدم الكاملة).
